### 동작 흐름 (변경 후)

```
사용자 입력
→ _append_message("user", ...)
→ _build_history()  ← 이전 Q&A 최대 3턴 추출
→ LangGraph (검색)
```

### 설계 결정

- **최대 3턴**: 토큰 예산 vs 문맥 필요량 균형. 환경변수로 조정 가능하도록 `MAX_HISTORY_TURNS` 상수화.
- **인용 블록 제거**: assistant 메시지의 `<!--RAG_REFERENCES-->` 이하를 히스토리에서 제거. JSON 인용 데이터가 토큰을 낭비하고 LLM 혼란 유발.
- **로컬 HF 동일 적용**: `apply_chat_template`이 멀티턴을 지원하므로 별도 처리 없이 동일 messages 배열 전달.

## 3-2 추가: GroundingCheck 강화 (문장 단위 인용 비율)

### 구현 내용

기존 `grounded = bool(re.search(r"\[\d+\]", ans))` 단순 ON/OFF에서
문장 단위 인용 비율 측정 + 수준별 경고 삽입으로 전면 재작성.

**변경 파일**

- **`graph/nodes/grounding_check.py`** 전면 재작성
  - `_grounding_score(ans)`: 의미 있는 줄(길이 > 10, 헤더·인용·마커 제외) 중 `[n]` 포함 비율
  - FULL (score > 0.5) → 경고 없음
  - PARTIAL (0 < score ≤ 0.5) → `ℹ 일부 문장에 인용 번호 없음` 삽입
  - NONE (score = 0) → `⚠ 회의록에서 인용 번호 확인 불가` 경고 삽입
  - 경고 삽입 조건: `docs 있음 AND ans.strip() AND 생성 정상`

- **`graph/state.py`** 신규 필드 2개 추가
  - `grounding_score: float` — 비율 (0.0~1.0)
  - `grounding_level: str` — `"FULL"` / `"PARTIAL"` / `"NONE"`

### 설계 결정

- **스트리밍 경로 보완**: LangGraph GroundingCheck는 비스트리밍에만 실행. 스트리밍 경로에서는 `_handle_user_input()`이 스트림 완료 후 동일 조건 재체크.
- **헤더·인용 줄 제외**: 마크다운 헤더(`#`)·blockquote(`>`)·bold 마커(`*`) 줄 제외 — 목록형 답변 오탐 방지.
- **임계값 0.5**: 절반 이상 인용이면 FULL. 더 엄격하면 모든 답변이 PARTIAL → UX 노이즈.

## Grounding Check 강화 2단계 — 완료 기준 구현

6개 완료 기준을 코드로 구현.

### 기준별 구현

| # | 기준 | 구현 위치 | 방식 |
|---|------|-----------|------|
| 1 | 주장마다 [n] | `prompt_templates.py` RULES_GROUNDING | "[n] 없으면 한계로" 규칙 삽입 |
| 2 | [n] 범위 검증 | `grounding_check.py` `_fix_out_of_range()` | 범위 밖 [n] → [?] 교체 |
| 2 | 버그 수정 | `generate.py` line 155 | `max_ref = len(citations)` → `len(docs)` |
| 3 | 청크 지지 체크 | `grounding_check.py` `_check_citation_support()` | 한국어 키워드 오버랩 휴리스틱 |
| 4 | 미인용 → 한계 | `grounding_check.py` `_move_uncited_to_limits()` | 세부 근거 미인용 불릿 이동 |
| 5 | 약한 검색 거부 | `grounding_check.py` `_is_weak_retrieval()` | MIN_DOCS=2, WEAK_SCORE=0.35 |
| 6 | 무근거 답변 거부 | `service/rag/eval/unanswerable_eval.py` | 10개 질문 평가셋 + 판정 스크립트 |

### 주요 로직

**`_is_weak_retrieval(docs)`**
```python
MIN_DOCS_FOR_ANSWER = 2      # 최소 문서 수
WEAK_SCORE_THRESHOLD = 0.35  # 최고 rerank_score 미만이면 weak
```
→ weak AND NONE → 답변 거부 (REFUSAL_WEAK 메시지)

**`_move_uncited_to_limits(ans)`**
- `## 세부 근거`의 [n] 없는 불릿(-) → `## 한계`로 이동 + `*(출처 미확인)*`
- `## 핵심 결론`의 미인용 문장 → 인라인 `*(출처 미확인)*` 표시

**`_fix_out_of_range(ans, num_docs)`**
- [n > num_docs] or [n < 1] → [?]

**generate.py 버그 수정**
- 기존: `max_ref = len(state.get("citations", []))` → citations 빌드 전이라 항상 0 → 무효
- 수정: `max_ref = len(docs)` (reranked 결과 수)

**RULES_GROUNDING (프롬프트)**
- 세부 근거 불릿에 [n] 없으면 한계로 이동 명시
- 검색 결과 2개 미만이면 답변 거부 형식 명시
