## E2E 통합 테스트

### 설계

ETL 재실행 없이 기존 DB 데이터로 전체 파이프라인을 검증하는 방향 선택.
PDF 재적재는 시간이 오래 걸리고 CI에 부적합하므로 기존 chunks/embeddings 데이터의 계약 검증 + RAG 파이프라인 통합에 집중.

### 구현 (`tests/conftest.py`, `tests/test_e2e_pipeline.py`)

**핵심 설계 결정:**
- `pg_port` 픽스처가 `PG_PORT` 환경변수를 설정하고 `update_db_config(DatabaseConfig.from_env())` 호출
- `Retriever` / `Generator` 임포트를 테스트 함수 내부에서 수행 (모듈 레벨 금지)
  - `config/vector_database.py`의 전역 `vector_db_config`가 임포트 시점에 초기화되므로, 픽스처가 환경변수를 먼저 설정한 뒤 임포트해야 올바른 DB 설정을 읽음
- DB 미연결 환경에서 자동 skip → CI 안전

**5단계 검증:**
1. `test_db_connection` — psycopg2 연결 + `SELECT 1`
2. `test_chunks_exist_and_schema` — chunk_id / text / metadata 계약
3. `test_embeddings_exist_and_dimension` — 벡터 차원 384 확인
4. `test_search_returns_results` — `Retriever.search()` → content / chunk_id / hybrid_score
5. `test_generate_with_citations` — `Generator.generate_with_citations()` → 비어있지 않은 답변

| 단계 | 검증 |

| 3 | embeddings_e5 테이블 레코드 수 |
